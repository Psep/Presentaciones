Antes que nada...
¿Qué es el Software Libre?

Software Libre, algo de historia...

  • Proyecto GNU:
    • Fundado en 1984 por Richard Stallman.
    • GNU es un acrónimo que significa "GNU No es Unix".
    • Consiste en crear un sistema operativo libre con todos sus componentes (intérpretes, compiladores, editores de texto, kernel, gestores de correo, etc).
    • El primer compilador(es) libre(s), GCC (GNU Compiler Collection).
    • Web: http://www.gnu.org

Software Libre, algo de historia...

  • Free Software Foundation:
    • Creada en 1985 por Richard Stallman.
    • Definición de Software libre y el concepto Copyleft.
    • Difusión del movimiento del software libre a nivel mundial.
    • Mantención de proyectos, licencias GPL, campañas de defensa y promoción de software libre.
    • Web: http://www.fsf.org

Las cuatro libertades del Software

  • La libertad para ejecutar el programa como usted desee y para cualquier propósito.
  • La libertad de estudiar cómo funciona el programa, y cambiarlo para que haga lo que usted quiera.
  • La libertad para copiar el programa y distribuirlo como usted guste.
  • La libertad para modificar el programa y distribuirlo con dichas modificaciones.

Copyright vs. Copyleft

El copyright no es un derecho natural sino un monopolio artificial impuesto por el Estado que limita el derecho natural de los usuarios a copiar.
Richard Matthew Stallman

GNU con Linux

  • A principios de la decáda del 90, el Proyecto GNU estaba casi completo con el sistema operativo, pero faltaba el kernel. El núcleo original del proyecto es Hurd.
  • En 1991, Linus Torvalds libera el kernel Linux bajo licencia GNU GPL. Reemplazo no comercial de MINIX.
  • Gracias a lo anterior, nace GNU/Linux, que es la combinación del kernel Linux con el Sistema GNU.

¿JEE? A lo nuestro...

¿Qué es JEE?

  • Significa Java Enterprise Edition.
  • Se rige bajo especificaciones y estándares abiertos bajo la Java Community Process (JCP).
  • APIs estándar (EJB, JDBC, Persistencia, JMS, etc).
  • Hoy en día, la plataforma líder a nivel empresarial.

JEE abierto, ¿Plataformas Privativas?

  • Existen variadas alternativas en el mercado, incluyendo las soluciones no libres.
  • Al tener licencia Apache, permite el software privativo.
  • Ejemplos como Oracle Middleware y Weblogic, IBM Websphere. Con sus IDEs respectivos como Oracle JDeveloper.
  • Buena parte del mercado está dominado por dichas marcas.

JEE abierto, ¿Plataformas Privativas?

  • ¿Por qué las soluciones no libres dominan el mercado?
    • Respaldo de grandes marcas.
    • Experiencia y madurez en sus plataformas (?)
    • Lobby y otras prácticas (?)
    • Soluciones empresariales enfocadas en SOA.

¿Qué es SOA?

  • Arquitectura Orientada a Servicios.
  • Paradigma enfocado a sistemas distribuidos y la abstracción.
  • Permite escalabilidad, flexibilidad e integración entre sistemas.
  • Abstracción de procesos y reglas de negocio como servicios.
  • Aborda situaciones reales como el encapsulamiento, desacoplamiento, reutilización.

Arquitectura SOA de Referencia

Arquitectura SOA de Referencia

  • Capa de Visualización: Interfáz hacia el usuario (mediante un servidor de aplicaciones).
  • Orquestación: Dirige los servicios. Estos son mediante BPM (y/o BPEL). Los procesos de negocio son abstraídos de las reglas de negocio (Motor de Reglas).
  • ESB: Bus de Servicio. Es donde se exponen y disponibilizan los servicios. El bus transforma y enriquece los datos.

Alternativas Libres

  • Capa de Visualización: JBoss AS (EAP), WSO2 Application Server, GlassFish*, Apache TomEE, Apache Gerónimo.
  • Orquestación:
    • BPM: JBoss BPM Suite, W2O2 Business Process Server.
    • Motor de Reglas: JBoss BRMS (integrado en BPM Suite), WSO2 Business Rules Server.
  • ESB: JBoss Fuse, WSO2 Enterprise Service Bus.
  • IDEs: Eclipse, JBoss Developer Studio, NetBeans, WSO2 Developer Studio.
